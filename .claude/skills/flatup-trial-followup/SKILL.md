# flatup-trial-followup

## 目的
体験者に圧をかけず、次の一歩を明確にする下書きを作る。送信は人間。

## タイミング
1. 体験前日リマインド（日時・持ち物・場所）— 雛形 `templates/trial_reminder.md`
2. 体験当日の案内
3. 体験後のお礼（当日〜翌日）— 雛形 `templates/trial_followup.md`
4. 翌日のフォロー
5. 入会案内（圧をかけない）
6. 口コミ依頼（満足確認後のみ）

## 原則
- 次の行動を1つに絞る。料金・日時は `canon/pricing_and_schedule.md` を確認。
- 文体は `canon/brand_voice.md`。医療効果・結果を断定しない。

## 禁止
- 圧の強い勧誘。自動送信。実績・数字の捏造。
