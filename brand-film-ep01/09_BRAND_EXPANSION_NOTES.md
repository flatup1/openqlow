# 成果物20: ホームページ・LINEボットへ転用する際の注意点

**前提: 本パッケージの段階では、ホームページとLINEボットの実装変更は行わない。** 広告の検証結果が出てから、ここに書いた手順と注意で転用する。

## 1. 転用の順序(正本§31)

1. 広告公開 → 検証(`08_TESTING_METRICS.md`)
2. 反応の良かった言葉・シーン・感情構造を特定する
3. **先にブランド正本を作る**(BRAND_CANON.md 等 — 正本§28の候補ファイル群)。HPやLINEを先にいじらない
4. 正本に基づいて、LINEボット・HP・SNS・接客文面へ一貫展開する

順序を守る理由: HPだけ先に変えると、LINEボットが従来の「格闘技寄り」のまま残り、ブランド体験が分裂する(正本§28の懸念)。**媒体より先に正本。**

## 2. ホームページへ転用するときの注意

- 広告で勝った要素だけを持ち込む。負けた要素(反応のなかったコピー・シーン)をHPで復活させない
- ヒーロー領域への転用第一候補: 60秒クリーン版(ナレーション入り)または S6/S9 の静止画+勝ちコピー
- 動画をヒーローに置く場合は、読み込み速度・自動再生のミュート仕様・モバイル通信量を必ず検証する(感情設計が速度低下で相殺されては本末転倒)
- 「四季構成案」(正本§30)はこの段階でも**実装しない**。正本策定時の提案として扱う
- 予約・料金などの実用情報を消さない。「短編作品として体験するサイト」への転換は、実用導線を壊さない範囲で段階的に行う
- コード変更はリポジトリを証拠ベースで分析し refactor-instructions.md を作ってから(正本§2)。ブランド転用を理由に実装AIが大規模書き換えをしない

## 3. LINEボット(AIKA)へ転用するときの注意

- ボットの人格を「夜の道具たち」にしない(広告の魔法は映像内の演出。ボットが道具を演じると安っぽくなる)。転用するのは**価値観と言葉遣い**のみ
- 転用する原則(正本§29): 売り込まない / 急かさない / 不安を否定しない / 初心者目線 / 強さより安全・楽しさ・続けやすさ / 体験予約をゴールにしすぎない / 金銭・退会・規約・健康・安全は正本参照 / 判断できないことは人間へ
- 広告で反応の良かったフレーズ(例: 「はじめの一歩」「見守る」)を、返信テンプレの語彙に反映する。ただし詩的なコピーをそのまま連発しない(会話では不自然)
- 既存の AGENTS.md のLINE運用規則(3〜5行・押し売りしない・送信は人間承認)と矛盾しないこと。矛盾したら AGENTS.md と正本が優先

## 4. SNS・接客文面への展開

- 投稿文・DM返信・店頭の声かけまで「先に共感 → 次に安心 → その後にFLATUP → 最後に優しい導線」の順序を統一する
- 禁止表現リスト(正本§27)は媒体を問わず適用する。特に「強い子に育てる」「いじめ対策」「今始めないと」系は接客トークでも使わない
- 第二話以降(親の背中編・女性の最初の一歩編など)は、道具キャラ3体(ミット/グロー/バッグさん)と `03_ART_BIBLE.md` の世界観設定を再利用して一貫性を保つ

## 5. ブランド正本を作るときに本パッケージから昇格させる項目

広告検証後、正本(BRAND_CANON.md 等)へ以下を移す:

- ブランドの核: 「人生の最初の一歩に寄り添う場所」「世界一初心者に優しい格闘技ジム」の使い分け(検証結果で主従を確定)
- 二層構造(第一話=親子 / ブランド全体=すべての初心者)
- 感情の順序(共感→安心→FLATUP→導線)
- 禁止表現リスト(正本§27)+検証で新たに見つかったNG
- 道具キャラ3体の設定(シリーズ資産として)
- ビジュアルアイデンティティ(白・木目・緑・蛍光ピンクのアクセント、優しい光)
- 検証で効いた実際の言葉(コメント引用つきで根拠を残す)
