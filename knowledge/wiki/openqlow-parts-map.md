# OPENQLOW 部品の地図（src フォルダ＝なに係？）

プログラムは `src/` の中で「係（フォルダ）」に分かれて仕事しています。
学校の**委員会**みたいなもの。だれが何の係かを表にしました。

| フォルダ | 中学生むけに言うと… | 係の名前 |
| --- | --- | --- |
| `generators/` | ネタや文章を**考えて作る** | 文章づくり係 |
| `crm/` | お客さんの**名簿を管理**する | 名簿係 |
| `line_bot/` | **LINEの窓口**。メッセージを受けて返す | 受付係 |
| `publish/` | 投稿の**下書きを準備・保管**する（いちばん大きい部屋） | 下書き準備係 |
| `distribution/` | 1つのネタを**各SNS向けに広げる** | 配り係 |
| `approval/` | 「OK／なおして／やめて」の**受付** | 承認うけ係 |
| `conversation/` | 朝の**聞き取り（インタビュー）**を進める | 聞き取り係 |
| `commands/` | 日報ToDoや月報など**便利な命令**いろいろ | 便利係 |
| `scheduler/` | **決まった時間に動かす**（時間割） | 時間割係 |
| `safety/` | あぶない投稿を**止める** | 安全見張り係 |
| `privacy/` | **個人情報**を守るルール | プライバシー係 |
| `monitor/` | ちゃんと動いているか**監視**する | 監視係 |
| `sources/` | **ぜったいに正しい事実**（料金など正本）を持つ | 正本（しょうほん）係 |
| `adapters/` | **外の道具とつなぐ**（Obsidian記録など） | つなぎ係 |
| `state/` | データを**ファイルに保存**する | 保存係 |
| `utils/` | 日付計算など**こまかい道具** | 道具箱 |

> いちばん大きい部屋は `publish/`（36ファイル）。投稿まわりの準備がとても多いということ。
> でも Phase 1 では「準備するだけ」で、**本当の投稿はしません**（→ [[openqlow-safety-rules]]）。

## 動く順番（かんたんな流れ）
1. **文章づくり係**がネタを作る
2. **配り係**が各SNS向けに広げる
3. **安全見張り係**がチェックする
4. **承認うけ係 / 受付係（LINE）**で人間が「OK」する
5. **下書き準備係**が下書きとして保存する（投稿はしない）
