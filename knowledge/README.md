# 🧠 knowledge — 第二の脳 (Second Brain)

あらゆる情報源（記事・書き起こし・PDF・メモ）を取り込み、Claude が整理して
`[[ウィキリンク]]` でつなぎ、**質問すれば答えが返る**生きているウィキにする仕組みです。

Andrej Karpathy が紹介した「AIをコードのためだけでなく“第二の脳”を作るために使う」
アイデアの実装です。情報を足すほど賢くなります。

## 3分でわかる仕組み

| フォルダ | 役割 |
| --- | --- |
| `sources/` | 生の入力を**そのまま**置く（原本。編集しない） |
| `wiki/` | Claude が整理して作る**ウィキページ**（1概念=1ページ、相互リンク） |
| `wiki/index.md` | すべての入口（目次 / Map of Content） |
| `templates/` | ソース・ウィキページの雛形 |
| `CLAUDE.md` | 全体を管理する運用ルール（Claude が従う司令塔） |

## 使い方

1. **入れる**: 取り込みたいテキストを `sources/` に置く、またはチャットに貼る。
2. **整理させる**: Claude に「**これを分析して整理して**」と言う。
   → Claude が原本を保存し、概念ごとにウィキページを作り、関連を繋ぎ、目次を更新します。
3. **引き出す**: 「**〇〇について保存した情報を教えて**」と質問する。
   → 保存済みの知識から、根拠リンク付きで答えます。

## Obsidian で開く場合

このフォルダはそのまま **Obsidian の Vault** として開けます
（`[[...]]` リンクが Obsidian のグラフ表示・バックリンクで生きます）。
`File → Open folder as vault` で `knowledge/` を選んでください。

## ルール（要点）

- `sources/` の原本は**書き換えない**。加工結果は必ず `wiki/` へ。
- APIキー・個人情報はここに**書かない**。
- 1ページ1概念。重複は作らず既存ページを更新する。

詳しい運用規約は [`CLAUDE.md`](./CLAUDE.md) を参照。
