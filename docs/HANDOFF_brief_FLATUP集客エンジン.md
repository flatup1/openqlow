# FLATUP GYM AI集客プロジェクト — 引き継ぎブリーフ（他AI用 / 完全版 v3）

> 目的：このプロジェクトでオーナー(JIN)が伝えてきた「やりたいこと」と「今どこまで出来ているか」を、抜けなく別AIに引き継ぐ。
> 大原則（北極星）：**中学生に一言で説明できないものは、作らない。**（`docs/SIMPLICITY.md`）

---

## 1. 背景・前提
- **FLATUP GYM**：千葉県成田市の格闘技ジム（キックボクシング中心、柔術あり）。
- **ターゲット**：初心者女性／キッズ／運動不足の社会人／自信をつけたい男性／シニア。
- **強み**：女性インストラクター在籍／初心者にやさしい／キッズ対応／初回体験500円／24時間サンドバッグ／専用駐車場／大会UIZIN／ガチスパー禁止で安全。
- **既存AI構成**
  - **AIKA**：顧客対応AI（LINE @jfl0054o・友だち971人）。返信・体験案内・FAQ。**自動送信は禁止**。返信は「AIKA」で締める。あいさつ文・リッチメニューはここの設定。
  - **openQLOW**：裏の経営・集客AI（オーナー専用LINE @817nsdhr）。日次分析・見込み客管理・改善提案。**自動送信禁止**。実体は **依存ゼロのTypeScript/ESM**（※Flask/Pythonではない）。

## 2. 最終ゴール
**FLATUP AI永久集客エンジン**＝集客を半自動で回すAI。
**顧客を常に体験につなげるために、学習・記録・改善を続ける永久機関**。
「問い合わせ→体験→入会→口コミ→再来店」を、AIが**営業参謀**として前に進める（営業担当ではない／**最終判断は必ず人間**）。

## 3. 絶対の禁止事項（厳守）
- AIが勝手に顧客へ送信／予約確定／料金変更しない
- AIが勝手に本番コードを書き換えない／大規模リファクタしない
- しつこい営業文・LINEブロックにつながる追客をしない
- 既存のAIKA／openQLOW／LINE webhookを壊さない
- APIキー直書きしない／個人情報を不要に外部送信しない
- **Python/Flask/SQLiteへ置き換えない**（正本はTS/ESM・依存ゼロ）

## 4. 進め方の希望（JINの作業スタイル）
- 既存構成を壊さない・**最小変更**・**差分のみ**追加
- **コードを書き換える前に「どこをどう変えるか」必ず聞く**
- **実機で動かして最適化**する
- 説明は**「中学生にもわかるように」**
- 重要判断は**4専門家チーム（①リスク②マーケ③経営④現場コーチ）→ ①リスク②構造③改善案④メリデメ⑤最適戦略⑥最小コスト**で分析
- 人間確認前提／ログを残す／あとで拡張しやすく
- **territory（担当領域）を厳守**（後述）

## 5. FLATUP GYM 正本データ（勝手に変えない・customer-facing）
- **体験は「初回体験500円」**（⚠「無料体験」は誤り・使わない）
- 2回目以降ビジター3,000円／キッズ7,700円／女性8,800円／男性9,900円／入会金10,000円
- 持ち物：動きやすい服・タオル・水／専用駐車場あり
- **体験できる種目：キックボクシング／柔術**（レスリングは体験種目に含めない）
- スケジュール：
  - キッズ＝火・木18:00、土13:00／レディース＝土14:00
  - **男性体験＝火・木・土（男性インストラクター在籍）、または平日19:00以降**
  - 女性体験＝月・水・土
  - 日曜・祝日は原則体験不可
- ガチスパー禁止／初心者・女性・キッズが安心して通えるジム
- 体験予約のトリガー文（あいさつ・リッチメニュー統一）＝**「体験を予約したいです」**
- 正本コードの場所：`src/generators/shared.ts` の `FLATUP_INFO`

## 6. AIKA返信ルール
- 丁寧・やさしい／初心者の不安を消す／強引に売り込まない／最後に自然に体験へ誘導／絵文字少なめ
- 料金を勝手に変更しない／日程を勝手に確定しない（候補日の確認にとどめる）
- 必要に応じて末尾「AIKA」
- **予約意思がある人**：質問を小分けにせず「①お名前 ②種目 ③希望日時」を一度に確認＋記入例（1往復で予約へ）
- **難条件の相談（途中参加など）**：希望種目を勝手に変えない・必要情報（到着時間/曜日）を聞く・代替案を出して会話を終わらせない
- **男性**：火・木・土（男性インストラクター在籍）＋平日夜を案内
