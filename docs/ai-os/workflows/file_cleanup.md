# ワークフロー：ファイル健診（削除しない）

対応する Skill：`flatup-file-audit`

## 手順

1. 対象ディレクトリを読み取り、次の問題を一覧化する。
   - 重複 / 古い料金 / 古い営業時間 / 未使用候補 / 名前の不統一 / 保存場所の誤り / 正本と矛盾する内容
2. 各項目に「なぜ問題か」「正本との差異」を添える。
3. **削除・移動は行わない。候補一覧までで停止する。**

## 原則

- 整理は「提案 → 承認 → 実行」の順（`canon/approval_matrix.md`）。
- 実際の削除・移動は人間承認後、別作業として行う。
- `rm -rf` / `git clean -fd` などは禁止（`approval_matrix.md` §3）。
