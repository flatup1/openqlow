# 外部連携（MCP）候補

> 現時点で存在しない MCP サーバーや架空の接続設定は作らない。
> 未接続の連携は「候補」としてここに記載する。認証情報はファイルへ直書きしない（環境変数名のみ）。

各連携の判断軸：
- 読み取り専用で始められるか
- 書き込み権限が必要か
- 個人情報を扱うか
- 人間承認が必要な操作
- API / MCP の有無
- 未接続時の代替手段

## 候補一覧

| 連携 | 読み取り開始 | 書き込み | 個人情報 | 人間承認が必要な操作 | 未接続時の代替 |
|------|:---:|:---:|:---:|------|------|
| GitHub | ○ | 要 | 低 | commit / push / PR / Issue変更 | ローカル文書で進捗管理 |
| Gmail | ○ | 要 | 高 | メール送信 | 下書きをテキストで作成 |
| Googleカレンダー | ○ | 要 | 中 | 予定登録 | 候補日時をテキストで提示 |
| Google Drive | ○ | 要 | 中 | ファイル書き込み・共有 | ローカルに保存 |
| Googleスプレッドシート | ○ | 要 | 高 | 会員・売上の書き込み | ローカル表を下書き |
| ブラウザ | ○ | 要 | 中 | 投稿・送信 | 手動投稿 |
| LINE公式アカウント | △ | 要 | 高 | 顧客への送信 | 返信下書きのみ |
| 会費ペイ | △ | 要 | 高（決済） | 請求・金銭処理 | 請求書下書きのみ |
| Instagram | △ | 要 | 中 | 投稿 | 投稿セット下書きのみ |
| Googleビジネスプロフィール | ○ | 要 | 低 | 投稿・返信 | 下書きのみ |

## 原則

- まず読み取り専用で接続し、書き込みは人間承認を挟む。
- 個人情報（会員名簿・健康情報・決済情報）を Git 管理下へ保存しない。
- 認証情報は要求せず、`.env` の環境変数名だけを記載する。
- 顧客への送信・投稿・金銭処理は、すべて下書きまで（`../canon/approval_matrix.md`）。
