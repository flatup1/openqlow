# 定時実行の方針

## 最初に自動化してよいもの

- 朝の司令書の下書き
- 週次KPI集計
- ファイル健診
- 正本との矛盾確認
- テスト結果確認
- 数値異常の検出

## 自動化しないもの

SNS投稿、LINE・メール送信、予約確定、料金・返金・退会判断、請求、顧客データ変更、本番反映。

## 候補

Codex Scheduled Tasks、Claude Codeの利用可能なスケジュール機能、launchd、cron、GitHub Actions、外部ワークフロー。採用前に、実行場所、読み取り範囲、秘密情報、失敗通知、停止方法、費用を確認します。

このセットアップでは定時実行を作成していません。まず手動実行で出力と承認境界を検証します。
