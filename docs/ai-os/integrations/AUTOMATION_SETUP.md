# 定時実行（自動化）候補

> いきなり自動投稿・自動返信を設定しない。最初に自動化してよいのは読み取り・集計だけ。

## 最初に自動化してよい作業（読み取り系）

- 朝の司令書作成（`flatup-daily-command`）
- 週次KPIレポート（`flatup-weekly-kpi`）
- ファイル健診（`flatup-file-audit`）
- 正本との矛盾確認
- テスト結果の確認
- 数値異常の検出（体験予約急減・退会増・決済エラーなど）

## 自動化してはいけない（下書きまで）

- SNS投稿 / LINE送信 / メール送信 → 下書き作成までで停止。

## 手段の候補（環境に存在するものだけ使う）

- Codex の Scheduled Tasks
- Claude Code のスケジュール機能
- cron
- launchd（本リポジトリに `launchd/` あり）
- GitHub Actions
- 外部ワークフローサービス

> 現在の環境に自動化手段が無い場合は、架空の設定を作らず、候補として記載するに留める。

## 参考（本リポジトリの既存資産）

- `launchd/`、`scripts/audit-scheduler.sh`、`docs/launchd.md` に既存のスケジュール実行資産がある。
  新規の定時実行は、まずこれらと整合するか確認する。
