# flatup-change-review

## 目的
変更を第二の目で検品し、事故を防ぐ。

## 確認項目
- 意図したファイルだけが変更されたか（`git diff --stat` を確認）
- 既存機能を壊していないか
- 正本（`canon/`）と矛盾していないか
- 個人情報・秘密情報（APIキー・トークン・会員情報）が含まれていないか
- テストが必要か
- 本番公開前に人間確認が必要か（`canon/approval_matrix.md` §2）

## 出力
- 変更の要約 / 懸念点 / 必要なテスト / 人間確認が必要かの判定。

## 禁止
- 検品を通したことにして commit / push / デプロイを自走させない。
