# LINE 文面テンプレ集（②③④⑤ / FLATUPトーン）

Bot が LINE で返す文言を FLATUP トーンで統一したテンプレ集。
Codex はこのまま `src/line_bot/` などで利用してよい。`{ }` は差し込み。
方針：短く・やさしく・JINが1タップで進める・誤解させない（投稿前は必ず確認）。

---

## ④ 朝の1候補確認

```text
おはようございます。今日はこの内容でいいですか？

本文:
{body}

投稿先: {destinations}

画像の候補（{media_dir} から）:
1. {file1}
2. {file2}
3. {file3}
→ 使う画像の番号を送ってください（例: 画像 1）。画像なしなら「画像なし」。

OK で進みます。直したいときは「修正 〇〇」、別の画像は「挿入」、
気に入った画像がなければ写真や動画をそのまま送ってください。
やめるときは「NO」。
```

### 候補なしの朝
```text
おはようございます。今日は出すのにちょうどいい候補がありませんでした。
無理に出さず、見送るのがおすすめです。
出したいネタがあれば「修正 〇〇」で本文から作れます。
```

### 画像を選んだ後（確定提示）
```text
こちらで進めますね。最終チェックをお願いします。

本文:
{body}
画像: {selected_file}
投稿先: {destinations}

OK / 修正 〇〇 / 挿入 / NO
```

---

## ② 修正

### 修正受付・再確認
```text
下書きを直しました。これでいいですか？

本文:
{new_body}

OK / さらに修正 〇〇 / NO
```

### safety に引っかかった時
```text
今の文だと、この部分が引っかかりました：{reason}
言い換えてもう一度「修正 〇〇」で送ってください。
（このままでは投稿に進めません）
```

### 対象が見つからない
```text
今、修正できる下書きが見当たりませんでした。
朝の候補を出すか、先に本文を作りましょうか？
```

## ③ 挿入（フォルダから選ぶ）

### 一覧提示
```text
{media_dir} の画像・動画はこちらです。
1. {file1}
2. {file2}
3. {file3}
使うものの番号を送ってください（例: 挿入 2）。
```

### 添付後・目視確認
```text
添付しました。中身を確認してください（映り込みに注意）。

本文:
{body}
画像: {selected_file}

OK / 修正 〇〇 / NO
```

### フォルダが空・不在
```text
{media_dir} に使える画像・動画が見つかりませんでした。
フォルダに入れるか、写真や動画をこのままLINEに送ってください。
```

### 非対応の形式
```text
この形式（{ext}）は今は使えません。
対応しているのは {allowed} です。
```

---

## ⑤ LINEから直接送られたメディア

### 受信・確認
```text
受け取りました（{saved_file}）。これを使いますか？
中身を確認してください（お客様・お子さんの映り込みに注意）。

本文:
{body}
画像: {saved_file}

OK / 修正 〇〇 / NO
```

### サイズ超過
```text
ファイルが大きすぎて受け取れませんでした（上限 {limit}）。
小さいサイズで送り直してください。
```

---

## 共通：OK のあと（①未解禁の半自動）

```text
OKですね。投稿の準備をします。
最後の投稿は、画面で内容を見て JIN が仕上げます（自動では出しません）。
```

> ①（完全自動）が解禁されるまでは「自動では出さない」ことを必ず一言添える。
> 「投稿しました」と言い切るのは、実際に投稿が確認できた時だけ。

---

## トーンの約束（全文面共通）

- 煽らない・盛らない・断定しない。
- 「投稿できていないのに成功」と読める言い方をしない。
- JIN が迷わないよう、選べる言葉（OK / 修正 / 挿入 / 画像 N / NO）を必ず添える。
