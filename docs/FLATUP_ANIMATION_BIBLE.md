## 0. 絶対ルール（全セクションに優先）

1. 既存IP（パワーパフガールズ等）のキャラクター・配色・衣装・構図・設定を使わない（ENGINE正本の禁止リスト準拠）
2. 「強さの誇示」「敵を倒す」を主題にしない。主題は常に **最初の一歩**
3. 初心者・女性・キッズが見て怖くない画面にする（暗くストイックな画面はNG。visual_brand.md 準拠）
4. 失敗シーンで登場人物は**絶対に笑わない・怒らない**。これは演出規約であり世界観の核
5. 完成映像の公開・広告利用は JIN 承認後のみ

## 1. キャラクターデザイン規約

- 主人公: フラッピー（仮称）。丸い輪郭・太陽マーク・大きめグローブ・「やってみたい」の表情
- 【TBD: Stage 2で確定】目の形／瞳サイズ／鼻と口／頭身／手足の長さ／グローブ形状／靴／ロゴ
- 確定後は `assets/character/flappy/reference/character_sheet.png` が唯一の正本。全生成はこれを IP-Adapter 参照にする

## 2. 色彩設計

- 基本パレット: 白 / 蛍光ピンク / 蛍光イエロー / グリーン（ジムの白床・緑マットと同系。visual_brand.md 準拠）
- 【TBD: Stage 2で確定】各色の正確なHEX値 → 確定後 `color_palette.png` + ここに色番号表を記載
- 夜のジムシーン: 暗黒ではなく「静かで温かい夜」。月光・常夜灯の暖色を残す

## 3. カメラワーク規約

- 基本: 子どもの目線の高さ。見下ろしの威圧アングルは失敗シーンで使わない
- 感情の瞬間（迷い・深呼吸・小さな笑顔）はクローズアップ + 静止で「間」を作る
- 【TBD】標準レンズ感・カット割りテンプレート（15秒版の絵コンテ確定時に追記）

## 4. ライティング規約

- 温かい照明を基本とする。逆光は「希望」の演出にのみ使う
- 音響設計と対応: 夜（静・青みの暖色）→ 朝日（テーマの完成）
- 【TBD】シーン別ライティング例（Stage 1 検証画像から採用例を貼る）

## 5. 演技（アクティング）規約

- 1動作 = 最低4段階（構え→予備動作→実行→余韻）。枚数より**感情が変わる瞬間**を優先
- 感情演技の標準シーケンス: 緊張 → 視線が泳ぐ → 呼吸 → まばたき → 拳を握る → 一歩踏み出す → パンチ → 安心した笑顔
- 失敗の演技: 転んだら「恥ずかしそうに周囲を見る」まで含めて1動作。すぐ立ち直らせない

## 6. 脚本ルール

- 感情曲線は必ず: 興味 → 共感 → 安心 → 挑戦 → 感動 → 行動（体験予約）
- 主人公は各話で「小さく1回だけ」成功する。連続成功・無双は禁止
- 結末は必ず「次の初心者へ手を差し伸べる」系の開かれた形にする
- 決め台詞（正本）:
  > 怖いままでも、大丈夫。
  > はじめの一歩を、笑わない。
- CTA文言: 「500円体験を予約する」（リンクは https://lin.ee/cTSDajPz ）

## 7. 音響規約

- 環境音優先: 夜のジム・時計・床・呼吸・革の音
- テーマ音楽は最後（感動→行動の転換点）でのみ完成させる
- 【TBD】テーマ音楽のモチーフ（girl-power-op の既存BGMエンジンを流用するか要判断）

## 8. 技術規約（Animation Engine との対応）

- ランタイム: `girl-power-op/`（PR #51 世代）。ポーズは `poses/` 固定名、シナリオは `story.js`
- 素材庫: `assets/character/flappy/`（reference / poses のグループ構成。ENGINE正本参照）
- 生成パイプライン: SDXL + FLATUP専用LoRA（`flatup_hero`）+ OpenPose + IP-Adapter
- 納品形式: 横 MP4 / 縦 1080x1920 MP4 / サイト埋め込みは `?loop=1` iframe
