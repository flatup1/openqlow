# 21 openQLOW週次運用手順（WEEKLY SOP）

作成: 2026-07-19 ／ 1週間の型。全体で**JINの手を1日5分、週30分**に収めることを目標にする。

## 月曜日に確認すること（10分）

1. 先週の数字を `20_OPENQLOW_WEEKLY_SCOREBOARD.md` の記録テンプレへ1行追加（4つだけでも可: 問い合わせ・体験実施・入会・退会申請）。
2. 警戒状態（2週連続）が出ていないかを段階1→5の順に見る。
3. 今週の予定を確認: 体験予約の入っている日／イベント（オープンデー・UIZIN）の準備物。
4. openQLOWに「今週の最重要1つ」を提案させ、JINが採否を決める。

## 毎日5分で確認すること

- 未返信のLINE問い合わせがないか（AIKA下書き→JIN承認→送信）。
- 明日体験の人がいればリマインド下書きを確認して送信（JIN操作）。
- 怪我・クレーム・退会・お金の話が来ていないか。来ていたら**その日のうちに**人間対応（`10_AIKA_SAFETY_CASEBOOK_50.md` 参照）。
- レッスンがあった日は写真を3〜5枚撮ったか（成長共有の材料）。

## 日曜日に振り返ること（15分）

1. 今週の「体験→入会」の結果と、入会しなかった人の理由をひとことメモ。
2. 成長共有LINEを送れたか。送れなかったなら理由（写真がない／時間がない）をメモ。
3. `23_FOUR_WEEK_EXPERIMENTS.md` の今週の実験の結果を記録（やった/やらない/数字）。
4. 来週やめること・続けることを1つずつ決める。

## 数字が悪い場合の判断順

1. **記録漏れではないか**（数えていないだけ、が最頻の原因）。
2. **入口から順に**見る: 問い合わせ→体験→入会→継続→口コミ→紹介。上流が細ければ下流は触らない。
3. 2週連続で悪い段階だけ `22_OPENQLOW_NEXT_ACTION_RULES.md` の判断表を引く。
4. 同時に変えるのは**1つだけ**。複数変えると何が効いたか分からなくなる。
5. 料金・キャンペーン・クラス編成に関わる対策は必ずJIN判断（AIは提案まで）。

## 仕事の分担

### AIKAへ渡す仕事（守り）
- 問い合わせ返信の下書き、体験案内、前日リマインド文、追客文（T14）、口コミ依頼文の下書き。
- 事故防止: ケースブックの分類に沿った一次返信と人間切替。

### openQLOWが担当する仕事（攻め）
- 週次数字の集計と警戒サイン提示、改善提案、投稿・配信・イベント告知の下書き、退会理由の傾向分析、実験の設計と評価。
- **やらないこと**: お客様への直接送信、自動返信、メール自動受信、予約確定、会員情報変更。

### 人間（JIN）だけが決める仕事
- すべての送信・配信、体験日時の確定、入会・退会・休会の成立、料金・割引・返金、会費ペイ操作、イベント日程、クレーム・怪我・法的対応の結論。

## 週次の記録場所

- 数字: スコアボードの表（このリポジトリ内 or JINの使いやすい場所。場所は一度決めたら動かさない）。
- 判断メモ: 同じファイルの「メモ」列に1行。長文にしない。
