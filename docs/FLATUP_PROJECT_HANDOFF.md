## 3. 作品の核となる感情設計

- **感情アーク**: 不安 → 希望 → 安心 → 小さな挑戦 → 喜び → 優しさの継承 → 行動(体験予約)。
- **中心テーマ「今しかない時間」**: 小さな手はいつか大きくなる。今この時間を一緒に見守れることは尊い。※親を罪悪感で追い込まない。
- **演出テーマ「親の背中を見て育つ」(伏線→回収)**: 説明せず、子が親を自然に真似る仕草(靴を履く背中を見る/しゃがむを真似る/礼を真似る)を無言で積み、最後にナレーション一言で回収。童話の温度。
- **世界観「夜のジムの語り」**: 夜、誰もいないジムで道具たち(ミット/グローブ/サンドバッグ)が、その日見た親子の成長を、技術でなく挑戦・思いやり・勇気について、ぽつぽつ語る。**セリフは固定せず原則だけ正本化 → シリーズ化の器**。
- **エモーションマーケ**: 人は感情で決め理性で正当化する。感情の変化を"言語化"して見せる(「最初は不安だった。でも、なんとかなった。」)。CTAはLINE友だち追加一択に絞る。

## 4. すでに作ったもの(実装・成果物)

### A. girl-power-op(動画レンダラ・完成/稼働中)
- HTML Canvasで「時刻tの絵」を描き、Playwrightで1コマずつ撮影→ffmpegでMP4化。BGMもWeb Audioでブラウザ内合成。**AIサービス不要・決定論的・サイト埋め込み(`?loop=1`)対応**。
- **`story.js`** にシーンを並べるデータ駆動。シーン型14種: welcome/entrance/punch(ジャブ→ストレート→フックの3連)/closeup/jump/sky/finale と、しずか系 night/fall/smallpunch/sunrise/help(優しさの継承)/rest(おやすみ)/cta(体験予約ボタン)。
- ChatGPTで作った設計資料(9ポーズ)から `poses/` を切り出し、キャラが実際に動くアニメを量産。ポーズ欠落時は `chara.png` へ自動フォールバック。
- 旧iOS Safari対策(roundRectポリフィル)・読み込み失敗の非致命化 済み。
- Colabノートブック(SDXL+IP-Adapter)でポーズ生成する導線も同梱。

### B. animation-studio(画像→動画Webアプリ・完成/デモ検証済み)
- React+TypeScript+Express。Google Veo(Gemini)で画像→動画生成。**デモモード同梱**でAPIキー無しでもUI検証可。APIキーはサーバー側限定。動作テンプレ12種/2〜10秒/9:16等/履歴/レート制限/zod検証。
- 実Veo接続は課金キーが必要で未検証(アダプタ実装済み)。**Seedance等への差し替えも VideoProvider 抽象化で容易**。
