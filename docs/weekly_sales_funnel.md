# 週次売上導線チェック表（毎週月曜・10分）

作成: 2026-07-19 ／ 対象: JIN（オーナー）
根拠正本: `flatup-ai-os/src/data/line_flows_100.md`（LINE文面・タグ運用）・`flatup-ai-os/docs/aika_safety_playbook.md`（守りの対応）・canon_2026（親子割・紹介キャンペーン）

**目的**: 体験予約 → 体験 → 入会 → 継続 → 口コミ → 紹介 の6段を、週1回・1枚で点検し、
「今週やるフォローを5件まで」に絞って実行する。全部やろうとしない。

---

## 1. 今週の数字（毎週月曜に埋める）

## 2. 判定と即アクション（迷ったらこの表に従う）

| 見つけた状態 | 判定 | やること（1本だけ） |
|--------------|------|-------------------|
| 体験予約が週0件 | 集客不足 | openQLOWで今週のSNS3案生成 → 1本投稿 |
| 体験→入会が2週連続0 | 転換の問題 | 体験当日の流れを1つ変える（お礼→翌日→3日後の3点セット確認） |
| 「体験済み未入会」が5人超 | フォロー滞留 | 古い順に3人へ `trial-followup` の文面で1本ずつ |
| 「2週間来ていない」が5人超 | 継続の問題 | クラス内容より先に「来やすさ」(時間帯・声かけ)を疑う |
| 口コミ依頼ゼロが3週続く | 資産づくり停止 | 今週レッスンで喜んでいた人を1人だけ選んで依頼 |
| 区分C(クレーム等)が週2件以上 | 仕組みの問題 | 個別対応で終わらせず、規定・掲示・クラス運営を1つ直す |

## 3. 使うコマンド（すべて下書き生成・送信は人間）

```bash
cd openqlow
npm run inquiry -- "<問い合わせ文>"                     # 問い合わせへの安全な一次返信下書き
npm run trial-followup -- --gender female --status 検討中  # 迷い中フォロー下書き
npm run daily                                           # SNS3案 → LINE承認へ
```

## 4. 週次の流れ（合計10分）

1. **数える(5分)**: §1の表を埋める（先週の列は前回の紙/メモから転記）
2. **選ぶ(2分)**: §2の判定表で、今週のアクションを**最大5件**選ぶ
3. **出す(3分)**: 下書きをopenQLOW/AIKAに作らせ、JINが直して送る
4. 埋めた表は Vault `01_DAILY_OPERATIONS/weekly/YYYY-MM-DD.md` に貼って保存（前週比較用）

## 5. やらないことリスト（重要）

- 週の途中で数字を何度も見ない（月曜だけ）
- 迷い中全員への一斉送信（1人1本・個別文面のみ）
- 口コミ・紹介の乱発（信頼の前借りになる）
- 数字が悪い週に施策を3つ以上始める（1つだけ変えて翌週測る）
