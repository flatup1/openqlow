# 投稿候補の採点基準（FLATUP / ④の1候補選定用）

朝フロー（④）で Bot が **複数案から最有力1案を選ぶ** ときの採点軸。
本文の良し悪し判定は FLATUP コンテンツ層（Claude 担当）の責任範囲。Codex はこの基準を
「最有力を選ぶロジック」に使ってよい。スコアの実装方法（重み付け・閾値）は Codex 判断。

> 目的：JIN が朝に見る1案が「FLATUP らしく、安全で、今日出す意味がある」ものになること。

---

## 1. 採点の5軸（各 0〜2点、満点10点）

| 軸 | 見るところ | 0点 | 1点 | 2点 |
|---|---|---|---|---|
| **A. FLATUPらしさ** | 価値提供・誠実さ・押し売り感のなさ | 煽り/誇大/値引き連呼 | 普通 | 体験価値が自然に伝わる |
| **B. 今日性** | 今日出す理由（季節/曜日/直近の出来事） | いつでもいい内容 | ややタイムリー | 今日出す意味が明確 |
| **C. 行動喚起** | 読んだ人が次に何をすればいいか | CTAなし | CTAあるが弱い | 体験/問い合わせ導線が自然 |
| **D. 読みやすさ** | 長さ・改行・一目で伝わるか | 長文/詰まり | 普通 | スマホで一目で入る |
| **E. 安全性** | 映り込み・断定医療効果・個人特定の無さ | リスク語あり | グレー | クリーン |

- **E が 0 点の案は採用しない**（安全はスコア合計より優先）。
- 同点なら B（今日性）が高い方を選ぶ。

---

## 2. 加点・減点の具体例

### 加点（FLATUPらしい）
- 「初心者・運動が苦手な人でも安心」など **心理的ハードルを下げる**表現
- お客様の成長や場の雰囲気を **具体的なシーン**で見せる（数字や情景）
- キッズ/格闘技の **健全で前向き**な切り口

### 減点（FLATUPらしくない）
- 「今だけ」「絶対」「最安」など **煽り・誇大・断定**
- 医療/減量効果の **断定**（「必ず痩せる」等）
- 他ジム・他者の **見下し/比較dis**
- 会員・お客様・子どもが **特定されうる**書き方

---

## 3. 画像候補の並べ方（④の画像提示順の参考）

本文採点とは別に、画像候補を出す順番の目安（Codex判断で利用）：

1. 本文の内容に合う被写体（キッズ練習の本文なら kids 系を上に）
2. 新しいファイル順
3. 映り込みリスクが低そうなもの（最終判断は必ず JIN の目視）

> 画像の中身の安全判定は自動では断定しない。必ず JIN の目視確認を残す（§映り込みルール）。

---

## 4. 「今日は出さない」を選べること

- 全案が E（安全）でグレー、または合計が低い朝は、**無理に1案を出さない**。
- 「今日は良い候補がありません」も正しい結果（④の仕様）。
- でっち上げて出すより、出さない方が FLATUP の信頼を守る。

---

## 5. Codex への引き渡しメモ

- このルーブリックは **選定の指針**。厳密なスコア式は Codex 実装で良い。
- 重み付けや閾値を変えたくなったら、本文の良し悪し基準は Claude 領域なので
  JIN 経由で Claude に相談（COORDINATION.md の領域分担）。
- 採点ログに **本文全文や個人名を不要に残さない**（必要なら recordId + スコアのみ）。
